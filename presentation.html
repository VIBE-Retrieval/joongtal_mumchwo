AI 운영 시스템 · 자기 개선형 루프

중탈멈춰.

국비 교육생의 중도탈락을 사전에 감지하고 자동으로 개입하는 AI 운영 시스템. 멘토 피드백을 학습해 다음 판단을 보정하는 자기 개선형 루프를 구현했습니다.

Next.js · FastAPI · MySQL Claude · scikit-learn 독립 실행 · LMS 연동 불필요
JOONGTAL_MUMCHWO / OVERVIEW
Problem

중도탈락은 늘 사후에 발견됩니다.

학생이 이미 포기한 뒤에야 멘토가 인지하는 구조. 면접 단계 위험 신호도, 과정 중 매일의 미세한 변화도, 그저 흘려보냅니다.

01 / 면접 단계
위험 신호를 흘려보냄
면접관의 직관에만 의존. 과거 중도탈락자의 패턴이 다음 기수 합격 판단에 활용되지 않음.
02 / 과정 중
사후 발견에 가까움
출결·점수가 무너진 뒤에야 인지. 매일 누적되는 미세한 하락 신호를 추적할 수단이 없음.
03 / 개입 이후
결과가 학습되지 않음
어떤 개입이 효과 있었는지 누적되지 않아 과잉 경보(false alarm)와 누락이 반복됨.
우리는 이 흐름을 거꾸로 뒤집습니다.
면접부터 위험을 예측하고, 매일 추적하고, AI가 직접 개입하고, 그 결과를 학습해 다음 판단을 보정합니다.

STACK / 03
Architecture

전체 AI 파이프라인

계층의 책임을 엄격히 분리합니다. ML은 계산만, LLM은 해석만, Agent는 결정만.

학생
오늘의 설문 응답 (성취도 / 적응도 / 인간관계, 각 1~5점) daily_survey 저장
↓
ML
최근 7일 평균을 4-7-1 신경망에 입력 → risk_score (0~1) risk_level / risk_trend 산출 · 숫자만 계산
↓
LLM
Claude가 위험도 점수를 자연어로 해석 state_summary · risk_reason · risk_type 생성
↓
Agent
LLM 해석 + 과거 개입 이력을 종합해 행동 결정 및 실행 NONE / ENCOURAGE_MESSAGE / ALERT_MENTOR / REQUEST_MEETING / EMERGENCY
↓
멘토
케어 완료 후 피드백 입력 → DB 누적 → Agent 보정 recovered / false_alarm / 효과 없음
PIPELINE / 04
Core 1

학습하는 AI Agent

"이전 개입의 결과가 다음 판단에 반영됩니다."
Feedback Loop
# 자기 개선형 루프 개입 실행 → 결과 저장 결과 저장 → 멘토 피드백 멘토 피드백 → Agent 보정 Agent 보정 → 다음 판단에 반영 # 예시 이전: REQUEST_MEETING → false_alarm 다음: ENCOURAGE_MESSAGE 로 한 단계 약화 이전: ALERT_MENTOR → recovered 다음: 해당 행동 점수 상승
Action Types · 강도순
NONE
개입 없음, 모니터링만
ENCOURAGE_MESSAGE
AI 생성 격려 메시지 전달
ALERT_MENTOR
멘토 대시보드에 위험 알림
REQUEST_MEETING
자동 면담 요청 생성
EMERGENCY
긴급 면담 + 멘토 즉시 알림
CORE FEATURE 1 / 05
Core 2

면접 단계 사전 선별 ML

"중도탈락자의 면접 패턴을 학습해, 입과 전에 위험 가능성을 예측합니다."
Inference Flow
면접 평가 (1~5점, 9개 항목) ↓ 12개 세부 feature + 파생 feature ↓ RandomForest 예측 ↓ dropout_risk_score (0 ~ 1) dropout_risk_level (LOW / MEDIUM / HIGH)

기존 수료자 / 중도탈락자의 면접 평가 결과로 RandomForest 모델을 학습. 면접관은 이 점수를 합격 / 보류 / 불합격 결정의 참고 지표로 활용합니다.

평가 3축
AXIS · 01
성취도
스스로 해내는 의지와 문제 해결 경험
AXIS · 02
학습 적응도
어려움에도 포기하지 않는 지속성
AXIS · 03
인간관계
팀 협력 및 고립 방지 역량
CORE FEATURE 2 / 06
Independence

어떤 LMS에도 연동이 필요 없습니다

대부분의 AI 교육 솔루션은 기존 LMS와 연동해야만 작동합니다. 이 시스템은 다릅니다.

01
별도 연동 없음
LMS · ERP · 기존 시스템과 연결하지 않아도 됩니다. API 키 발급도, 데이터 이전도 필요 없습니다.
02
설치 후 즉시 운영
pip install · npm install · 서버 실행. 끝.
03
데이터는 자체 보유
학생 데이터가 외부 플랫폼으로 나가지 않습니다. 모든 운영 기록은 내부 DB에 보관됩니다.
04
커스터마이징 자유
소스 코드가 전부 공개되어 있어, 운영 정책에 맞춰 행동·임계치·메시지를 자유롭게 수정할 수 있습니다.
기존 운영 방식을 그대로 유지하면서, 이 시스템만 옆에 붙이면 됩니다.
INDEPENDENCE / 07

DATA / 08
How we built it

개발 방법론

코딩을 먼저 시작하지 않았습니다. 설계를 먼저 끝냈습니다.

PHASE · 01
논문 기반 설계
중도탈락 예측 선행 연구를 분석해 3축 평가 모델과 이중 예측 구조(면접 + 과정)를 설계. 4-7-1 신경망과 RandomForest 모두 검증된 접근법을 적용.
PHASE · 02
API 설계 + 계층 정의
구현 전 API 명세, DB 스키마, ML feature, Agent 행동 정책을 모두 docs/ 에 코드보다 먼저 정의. 이후 모든 구현의 기준.
PHASE · 03
3시간 만에 전체 구현
계층 설계와 API 명세가 완성된 상태에서 코딩 시작 → 면접 ML → 과정 ML → LLM → Agent → 피드백 루프까지 단 3시간.
TOOLING
Claude Orchestrator 패턴
AGENTS/ 디렉토리에 도메인별 전문 에이전트 지침 정의. 오케스트레이터는 직접 코드를 작성하지 않고, 영향 범위를 분석한 뒤 적절한 에이전트에게 라우팅.
DB → Backend → AI → Frontend → Docs
CONTEXT EFFICIENCY
code-review-graph로 토큰 절감
전체 파일을 매번 읽지 않고, MCP 그래프로 함수 호출 관계와 영향 범위(impact radius)를 즉시 파악. 대규모 코드베이스에서도 필요한 컨텍스트만 정확히 로드.
METHODOLOGY / 09
Principles

설계 원칙

전체 시스템을 관통하는 다섯 가지 원칙.

PRINCIPLE · 01
ML은 숫자만 계산한다
해석은 LLM, 결정은 Agent. 각 계층은 자기 책임 밖을 침범하지 않는다.
PRINCIPLE · 02
모든 것은 기록된다
판단·개입·결과·피드백 전부 DB에 저장. 흔적이 없으면 학습도 없다.
PRINCIPLE · 03
데이터가 Agent를 개선한다
피드백 루프가 작동하지 않으면 Agent는 정체된다. 멘토 피드백이 곧 다음 판단의 근거.
PRINCIPLE · 04
문서 = 코드
docs/는 AI 판단의 기준이며 항상 최신 상태를 유지한다.
PRINCIPLE · 05
연동 없이 독립 실행
외부 시스템 의존 없이 설치만으로 운영 가능. 데이터는 자체 보유, 커스터마이징 자유.
